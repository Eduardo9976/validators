# 🌐 Sistema de i18n

## Como funciona

O sistema de i18n é baseado em **code + params** — sem mensagens hardcoded em nenhuma camada.

Cada erro produz um `code` (ex: `minLength`) e um objeto `params` (ex: `{ min: 5, actual: 2 }`).  
A função `translate(code, params, locale)` resolve a mensagem final via template de string.

### Fluxo

```
Erro ocorre → { code: "minLength", params: { min: 5, actual: 2 } }
                         │
                         ▼
         translate("minLength", { min: 5, actual: 2 })
                         │
                         ▼
         Template: "Minimum length is {min} characters, got {actual}"
                         │
                         ▼
         Resultado: "Minimum length is 5 characters, got 2"
```

### Cadeia de fallback

```
locale solicitado → "en" → código em si (nunca quebra)
```

## API

### `setLocale(locale)`
Muda o locale globalmente para todas as validações.
```js
const { setLocale } = require('validators')
setLocale('pt-BR')
```

### `getLocale()`
Retorna o locale ativo.
```js
getLocale() // → "pt-BR"
```

### `translate(code, params, locale?)`
Traduz manualmente um código de erro.
```js
translate('minLength', { min: 5, actual: 2 })
// → "Minimum length is 5 characters, got 2"

translate('minLength', { min: 5, actual: 2 }, 'pt-BR')
// → "O comprimento mínimo é 5 caracteres, recebido 2"
```

### `registerLocale(locale, messages)`
Registra ou mescla um novo locale em runtime.
```js
registerLocale('fr', {
  required: 'Ce champ est requis',
  email: 'Doit être une adresse e-mail valide',
  minLength: 'Longueur minimale : {min} caractères, reçu {actual}',
})
```

### `hasLocale(locale)`
Verifica se um locale está registrado.
```js
hasLocale('fr') // → true após registerLocale
```

---

## Override por validação específica

Você pode sobrescrever o locale para uma única chamada sem afetar o estado global:

```js
// Locale global: 'en'
validate(schema, value, { locale: 'pt-BR' }) // usa pt-BR só aqui
validate(schema, value)                       // volta a usar 'en'
```

## Adicionando um novo idioma

```js
const { registerLocale } = require('validators')

registerLocale('es', {
  required: 'Este campo es obligatorio',
  type: 'Se esperaba tipo "{expected}", se recibió "{actual}"',
  minLength: 'La longitud mínima es {min} caracteres, se recibieron {actual}',
  maxLength: 'La longitud máxima es {max} caracteres, se recibieron {actual}',
  email: 'Debe ser una dirección de correo electrónico válida',
  url: 'Debe ser una URL válida',
  pattern: 'El valor no coincide con el patrón requerido',
  notEmpty: 'El valor no puede estar vacío',
  min: 'El valor debe ser al menos {min}',
  max: 'El valor debe ser como máximo {max}',
  integer: 'El valor debe ser un número entero',
  positive: 'El valor debe ser un número positivo',
  negative: 'El valor debe ser un número negativo',
  minItems: 'Se requieren al menos {min} elementos, se recibieron {actual}',
  maxItems: 'Se permiten como máximo {max} elementos, se recibieron {actual}',
  noEmpty: 'El arreglo no puede estar vacío',
  unique: 'Los elementos del arreglo deben ser únicos',
  requiredKeys: 'Claves obligatorias faltantes: {keys}',
  oneOf: 'El valor debe ser uno de: {values}',
  custom: 'La validación falló',
})

setLocale('es')
```

---

## Mescla parcial

Você pode sobrescrever apenas algumas mensagens de um locale existente:

```js
// Apenas customiza a mensagem de 'email' em inglês
registerLocale('en', {
  email: 'Please enter a valid work email address',
})
```

## Placeholders disponíveis por código

| Código | Placeholders |
|---|---|
| `type` | `{expected}`, `{actual}` |
| `minLength` | `{min}`, `{actual}` |
| `maxLength` | `{max}`, `{actual}` |
| `pattern` | `{pattern}` |
| `min` | `{min}` |
| `max` | `{max}` |
| `minItems` | `{min}`, `{actual}` |
| `maxItems` | `{max}`, `{actual}` |
| `requiredKeys` | `{keys}` |
| `oneOf` | `{values}` |
